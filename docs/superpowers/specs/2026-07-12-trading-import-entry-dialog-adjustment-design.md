# 交易管理三表导入入口与弹窗调整设计

## 目标

修正交易管理 P0 的三表导入入口范围和导入弹窗反馈，不改变既有后端导入接口、三表齐全校验、预检、覆盖规则及事实数据模型。

## 入口范围

- 当前阶段只在“交易管理 / 持仓与交易”页面显示“导入三表”入口。
- 总览、上海钧能台账、期权台账、汇总与导出均不显示导入入口。
- 后端继续只保留现有一组预检和确认接口，不复制接口或导入逻辑。
- 未来“汇总与导入”页面方案确认后，只迁移同一个前端入口。

## 弹窗布局

- 宽度应用到外层 `#tradingImportDialog`，不再让内部表单宽度突破外层弹窗。
- 弹窗使用单列布局，完整显示账户、成交记录、平仓记录、期末持仓、预检结果和操作按钮。
- 弹窗宽度不超过当前视口，最大高度不超过视口；内容超高时仅弹窗内部纵向滚动。
- 小屏幕继续保持单列，不产生左右分栏或横向裁切。

## 处理状态

- 弹窗内增加独立状态区，依次支持：待选择、正在预检、预检通过、正在覆盖导入、导入失败。
- 预检期间显示加载动画和“正在预检三表，请稍候”。
- 确认期间显示加载动画和“正在覆盖导入并建立事实匹配，请勿关闭窗口”。
- 处理中禁用账户、三个文件、取消、预检和确认按钮，防止关闭和重复提交。
- 确认按钮在预检成功前隐藏；预检成功后显示。
- 导入成功后关闭弹窗、清理当前预览状态、刷新当前页面并在页面工具栏显示导入结果。
- 导入失败时弹窗保持打开，在弹窗状态区显示错误，并恢复可操作状态；页面工具栏同步显示失败。

## 状态重置

- 每次打开弹窗时清理上次预览批次、文件选择、预检结果和错误状态。
- 预检成功后，如果用户修改账户或任一文件，旧预览立即失效，确认按钮重新隐藏，必须再次预检。

## 测试与验收

- 前端契约测试确认导入入口只在 `trading_positions` 激活时显示。
- 前端契约测试确认宽度设置在外层弹窗、弹窗可纵向滚动且表单为单列。
- 前端契约测试确认预检和确认都进入 loading、禁用控件，并在 `finally` 中恢复。
- 本地浏览器验证五个交易管理菜单的入口可见性、完整弹窗、预检状态及确认 loading。
- 全量 Python、Node 测试通过后推送 Staging，并在 Render Staging 复验静态资源和控制台。

## 非目标

- 本次不迁移入口到“汇总与导出”。
- 本次不改变后端导入协议、解析和覆盖逻辑。
- 本次不开发导出能力、浮动盈亏或期权风险指标。
