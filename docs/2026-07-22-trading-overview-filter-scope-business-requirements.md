# 交易管理总览筛选、统计口径与明细入口优化业务需求说明

## 1. 文档信息

- 文档状态：业务口径与 Gate A 已确认，Staging 开发验证中
- 适用模块：交易管理 / 总览、交易管理 / 持仓与交易（本次更名为“持仓与交易明细”）
- 首次实施环境：`staging` 分支、Render Staging 服务 `ltm-web-staging`、Supabase `LTM WEB STAGING`
- 关联设计：`docs/superpowers/specs/2026-07-12-trading-management-design.md`
- 本文只定义本次总览筛选、统计口径、显示语义、性能和明细入口调整，不修改原始交易事实、既有业务归属结果或真实交易安全边界。

## 2. 业务背景

当前交易管理总览已经展示期间成交、平仓盈亏、手续费、期末持仓、期末保证金和逐日盈亏趋势，并在测试版增加“全部 / 基础套保 / 战略套保”切换。但现有页面存在以下业务问题：

1. 总览首次加载和筛选切换耗时明显，用户无法快速判断筛选是否已经生效。
2. 自定义日期提交后输入框清空，已应用日期范围没有稳定回显。
3. 月、季和自定义范围的期末持仓可能因为结束日没有同日快照而错误显示为 0。
4. 连续切换筛选时，较早发出的慢请求可能覆盖较晚选择，造成按钮口径与展示数据不一致。
5. “全部”与“基础套保 / 战略套保”实际使用不同口径，但页面没有清晰说明“全部”为事实口径、另外两项为业务口径。
6. 当前业务口径总览容易把“按业务归属筛选的事实盈亏”与“业务归属盈亏”混用。
7. 顶部账户和交易日期控件未形成完整、统一、可验证的筛选链路。
8. “持仓与交易”实际承担事实明细查询、核验和归属管理职责，现有名称不能充分表达页面用途。

本次修改需要先保证统计范围和业务口径正确，再优化查询性能和页面显示；性能优化不得改变业务语义或隐藏未归属数据。

## 3. 业务目标

本次修改需要实现以下目标：

1. 总览明确提供“全部、基础套保、战略套保”三种统计范围。
2. “全部”使用事实口径，盈亏显示事实盈亏；“基础套保、战略套保”使用业务口径，盈亏只显示业务归属盈亏。
3. 三种统计范围的成交、手数、手续费、持仓、保证金和趋势均随所选口径同步变化。
4. 三种统计范围统一支持日、月、季和自定义日期范围筛选。
5. 期间指标与时点指标使用明确且一致的日期规则，结束日没有持仓快照时取结束日前最近有效快照，不以 0 冒充无快照。
6. 筛选状态、页面显示和接口结果始终一致，旧请求不得覆盖当前选择。
7. 在当前 Staging 数据规模下显著缩短总览查询时间，并提供即时加载反馈。
8. 将“持仓与交易”更名为“持仓与交易明细”，作为事实明细查询和核验入口。

## 4. 修改范围

### 4.1 本期包含

- 总览统计范围切换及事实/业务口径命名；
- 日、月、季、自定义日期选择和已应用范围回显；
- 账户筛选的真实查询接入；
- 成交、手续费、盈亏、持仓和保证金的统一范围计算；
- 事实盈亏与业务归属盈亏的分离；
- 结束日前最近有效持仓快照规则；
- 未归属数据的质量提示；
- 筛选请求并发控制、加载状态和错误状态；
- 总览专用汇总查询与必要的查询性能优化；
- “持仓与交易”菜单、标题、面包屑和说明文字更名；

- 不新增“事实口径”第四个切换按钮；
- 不在基础套保或战略套保页面同时展示事实盈亏；
- 不新增未归属业务页签；
- 不修改基础套保、战略套保的归属结果、开平分配或历史审计；
- 不新增业务类型、业务主体或策略；
- 不修改原始交易事实、结算单导入、事实版本和来源追溯规则；
- 不开发汇总导出、浮动盈亏或新的期权风险指标；
- 不修改其它业务模块；
- 不操作 `main`、Production、正式数据库、正式环境变量或正式业务数据。

## 5. 业务术语

- **事实口径**：直接基于当前有效交易事实、平仓事实和持仓快照统计，不受业务归属或业务开平重配影响。
- **业务口径**：只统计已经归属到指定业务类型的交易和持仓，并使用业务开平关系计算业务归属盈亏。
- **事实盈亏**：当前有效平仓事实记录的逐笔平仓盈亏汇总。
- **业务归属盈亏**：按业务归属及当前有效业务开平关系计算并归属于基础套保或战略套保的盈亏。
- **期间指标**：在开始日至结束日内累计的成交、手数、手续费、盈亏和趋势。
- **时点指标**：截至结束日某一有效快照时点的持仓和保证金。
- **最近有效快照**：快照日期小于或等于筛选结束日的最新一份当前有效持仓快照。
- **未归属数据**：尚未归属基础套保或战略套保的事实交易及其尚未完成业务分配的关联结果。

## 6. 统计范围与盈亏口径

### 6.1 三种统计范围

总览保留以下三个切换项，顺序固定：

1. `全部`
2. `基础套保`
3. `战略套保`

规则如下：

- `全部`代表事实口径，包含全部当前有效事实，包括尚未完成业务归属的数据。
- `基础套保`只包含归属 `basic_hedging` 的业务数据。
- `战略套保`只包含归属 `strategic_hedging` 的业务数据。
- 未归属数据只进入“全部”，不进入基础套保或战略套保。
- 切换统计范围后，期间成交、成交手数、手续费、盈亏、期末持仓、期末保证金和趋势图必须全部使用同一统计范围。
- 三种统计范围不得复用前一次请求结果冒充当前结果。

### 6.2 全部口径

选择“全部”时：

- 期间成交、成交手数和手续费来自当前有效成交事实；
- 盈亏指标名称固定为“事实盈亏”；
- 盈亏数值来自当前有效平仓事实，不受业务归属和业务开平重配影响；
- 逐日趋势名称为“逐日事实盈亏趋势”；
- 期末持仓和期末保证金来自最近有效事实快照；
- 页面显示“事实口径”标签。

### 6.3 基础套保与战略套保口径

选择“基础套保”或“战略套保”时：

- 成交记录数、成交手数和手续费只统计归属该业务类型的部分；
- 盈亏指标名称固定为“业务归属盈亏”；
- 盈亏数值必须使用当前有效业务开平关系计算，不得使用按业务类型过滤后的事实盈亏代替；
- 发生人工业务开平重配后，业务归属盈亏随有效业务关系变化，事实盈亏保持不变；
- 逐日趋势名称为“逐日业务归属盈亏趋势”；
- 业务持仓按截至结束日的已归属开仓减去业务平仓分配计算；
- 业务保证金按对应业务持仓在最近有效事实快照中的持仓比例分配；
- 页面显示“业务口径”标签；
- 页面不同时展示事实盈亏或事实盈亏对照卡片。

### 6.4 记录数与持仓条数

- 期间成交记录数以当前统计范围内的稳定成交事实身份计数。
- 一笔事实记录同时涉及多个业务分配时，各业务范围只计算属于本范围的有效分配，不把整笔数量和手续费重复计入。
- 总览“期末持仓”条数统一按“合约 + 方向 + 资产类型”聚合后的非零持仓组数展示。
- 原始来源行数和聚合前记录数只在“持仓与交易明细”中查看，不作为总览主指标。

## 7. 时间筛选规则

### 7.1 时间维度

总览统一支持以下四种时间维度：

- **日**：选择一个具体日期，开始日和结束日均为该日。
- **月**：选择一个自然月，范围为该月第一日至最后一日。
- **季**：选择年份和自然季度，范围为该季度第一日至最后一日。
- **自定义**：选择开始日期和结束日期，包含首尾日期。

每种时间维度都必须提供真实可操作的选择控件，不得只显示按钮并隐式使用上一次返回结果推导日期。

### 7.2 默认范围

- 首次进入总览默认选择“月”。
- 默认月份取系统当前有效交易事实中的最新业务日期所在自然月。
- 首次请求必须携带实际开始日和结束日，页面不得显示“月”但返回全历史数据。
- 切换日、月、季或自定义后，保留当前统计范围，不自动切回“全部”。

### 7.3 指标日期字段

- 成交记录数、成交手数和手续费按 `trade_date` 落入筛选范围统计。
- 事实盈亏按事实 `close_date` 落入筛选范围统计。
- 业务归属盈亏按业务平仓对应的 `close_date` 落入筛选范围统计。
- 逐日趋势按相应盈亏口径的 `close_date` 聚合。
- 开始日只作用于期间指标，不限制期末持仓的历史开仓来源。
- 期末持仓和期末保证金按筛选结束日取最近有效快照。

### 7.4 快照规则

- 快照日期必须满足 `snapshot_date <= 筛选结束日`，并取其中最大日期。
- 结束日是周末、节假日、未来日期或没有同日快照时，不得要求快照日期与结束日完全相等。
- 找到最近有效快照时，页面显示“持仓截至：实际快照日期”。
- 没有任何结束日前快照时，显示“无可用持仓快照”，持仓和保证金显示为不可用状态，不显示为业务上的 0。
- 真实持仓数量为 0 时显示 0，并明确区别于“无可用持仓快照”。

### 7.5 已应用范围回显

- 页面必须持续显示当前实际应用的开始日、结束日和持仓快照日。
- 自定义日期提交并返回结果后，两个日期输入框必须保留已应用值。
- 月和季必须使用真实月末、季末日期，不得固定拼接不存在的 31 日。
- 开始日期晚于结束日期时，前端阻止请求并显示明确提示。
- 用户修改控件但尚未点击应用时，当前已展示数据仍标注原已应用范围，避免把待应用值误认为已生效。

## 8. 账户筛选

- 顶部账户筛选必须真实参与总览查询，不能保留为装饰性控件。
- “全部账户”统计用户有权查看的全部交易账户。
- 选择指定账户后，三种统计范围和四种时间维度全部只统计该账户数据。
- 账户、统计范围和时间范围共同组成一次完整筛选条件。
- 页面和接口必须返回当前实际生效的账户范围。
- 本期不修改账户权限模型；用户仍只能查询现有权限允许查看的账户。

## 9. 页面显示与交互

### 9.1 筛选区

- 第一组为时间维度：日、月、季、自定义。
- 第二组为统计范围：全部、基础套保、战略套保。
- 时间选择控件随时间维度变化，并在同一区域显示。
- 页面显示一条已应用范围摘要，例如：`统计期间：2026-07-01 ～ 2026-07-31｜持仓截至：2026-07-21｜全部账户`。
- “全部”旁显示“事实口径”；基础套保和战略套保显示“业务口径”。

### 9.2 指标卡片

总览保留五项主指标：

1. 期间成交；
2. 期间盈亏；
3. 期间手续费；
4. 期末持仓；
5. 期末保证金。

显示规则：

- “全部”下第二项名称为“期间事实盈亏”。
- 基础套保和战略套保下第二项名称为“期间业务归属盈亏”。
- 每张卡片的说明文字必须说明数据来源或时点，不使用容易误解的统一占位文案。
- 期末持仓卡片同时显示实际快照日期。
- 无数据、无快照、未完成业务归属和真实数值 0 必须使用不同显示状态。

### 9.3 趋势图

- “全部”展示逐日事实盈亏趋势。
- 基础套保和战略套保展示对应业务类型的逐日业务归属盈亏趋势。
- 趋势图必须使用与指标卡片相同的统计范围和日期范围。
- 没有盈亏记录时显示“当前范围暂无盈亏数据”，不得保留上一次趋势。
- 图表点的日期、盈亏值和合计必须能与同口径明细或数据库查询核对。

### 9.4 数据质量区

数据质量区使用当前账户和日期范围下的真实状态，不使用固定“已确认”文案，至少展示：

- 当前有效成交事实状态；
- 当前有效平仓及期权了结状态；
- 最近有效持仓快照日期；
- 未归属成交数量；
- 尚未完成业务平仓分配的数量；
- 当前筛选范围是否存在缺失快照或不完整业务归属。

未归属数据只作为质量提示，不新增总览切换页签。

### 9.5 明细入口更名

- 左侧菜单“持仓与交易”更名为“持仓与交易明细”。
- 页面标题、面包屑、权限配置显示名称和相关说明文字同步更名。
- 模块编码和现有权限能力保持不变，不新增权限编码。
- 页面副标题改为“查询和核验全部成交、平仓及持仓事实”。
- 总览中的明细跳转入口统一指向“持仓与交易明细”。

## 10. 加载、并发与错误处理

- 点击统计范围、切换时间维度或应用日期后，页面应在 200 毫秒内显示明确的局部加载反馈。
- 加载新结果时保留当前已展示数据和已应用范围，不整页清空或闪白。
- 同一总览同时只能有一个结果具备最终回写资格；旧请求即使更晚返回，也不得覆盖当前选择。
- 返回结果必须携带或能够校验账户、统计范围、开始日和结束日，前端只渲染与当前选择一致的结果。
- 请求失败时保留上一次成功结果，并明确提示本次筛选未生效。
- 请求失败、无数据和无快照必须分别显示，不得统一变成 0。
- 连续点击相同筛选不得产生无上限重复请求。

## 11. 性能要求

- 总览使用专用汇总查询，不得为了生成五张指标卡和一条趋势图而调用或拼装多个明细分页结果。
- 汇总查询只读取生成当前结果所需的聚合字段，不返回明细行、抽屉字段或无关策略列表。
- 在 Render Staging 服务已处于运行状态、使用当前 Staging 数据规模时：
  - 普通日、月、季和自定义筛选从操作到结果稳定显示应不超过 2 秒；
  - 基础套保和战略套保切换从操作到结果稳定显示应不超过 2 秒；
  - 任一正常筛选超过 3 秒视为性能验收失败。
- Render 冷启动时间单独记录，不计入汇总查询性能，但冷启动后第一次业务查询仍需记录实际耗时。
- 查询优化必须以实际执行计划和真实 Staging 数据验证，不得通过缩小默认业务范围、删除指标或隐藏未归属数据换取速度。

## 12. 异常场景

| 场景 | 页面处理 | 业务结果 |
| --- | --- | --- |
| 自定义开始日晚于结束日 | 阻止请求并提示日期范围错误 | 保留上一次成功结果 |
| 结束日没有同日持仓快照 | 取结束日前最近有效快照 | 显示实际持仓截至日期 |
| 结束日前没有任何快照 | 显示“无可用持仓快照” | 不把持仓和保证金显示为 0 |
| 当前范围没有成交或平仓 | 对应期间指标显示 0，趋势显示暂无数据 | 不沿用上一次结果 |
| 基础/战略范围没有已归属数据 | 对应业务指标显示 0 | 明确为真实空业务范围 |
| 存在未归属交易 | 只进入“全部”，数据质量显示数量 | 不进入基础或战略 |
| 业务保证金无法与事实快照匹配 | 显示保证金不可用及质量提示 | 不伪造为 0 |
| 用户快速连续切换筛选 | 只接受最后一次选择的结果 | 旧请求不得覆盖 |
| 汇总接口失败 | 保留上次成功结果并提示未更新 | 不改变已应用范围标识 |

## 13. 验收标准

### 13.1 口径验收

- “全部”只显示事实口径指标和事实盈亏。
- “基础套保、战略套保”所有指标均按对应业务归属统计，盈亏只显示业务归属盈亏。
- 人工调整业务开平关系后，基础或战略业务归属盈亏按新关系变化；“全部”的事实盈亏保持不变。
- 未归属数据进入“全部”，不进入基础套保或战略套保。
- 三种统计范围的成交、手数、手续费、持仓、保证金和趋势使用相同范围条件。

### 13.2 时间验收

- 日、月、季、自定义均能选择真实范围并驱动查询。
- 首次进入显示“月”时，请求和结果确实只覆盖默认月份。
- 自定义日期返回后输入值不清空。
- 月末、季末、周末和节假日都能取得结束日前最近有效持仓快照。
- 页面明确显示统计期间和实际持仓截至日期。
- 无快照与真实持仓 0 的显示不同。

### 13.3 交互验收

- 连续快速切换“全部、基础套保、战略套保”，最终数据始终对应最后一个激活项。
- 连续切换时间范围时，旧请求不能覆盖当前结果。
- 加载期间保留原结果并出现局部加载提示。
- 接口失败时保留原结果并提示本次筛选未生效。
- 账户筛选能够真实改变汇总结果，并与统计范围、时间范围共同生效。

### 13.4 显示验收

- “全部”显示“事实口径、期间事实盈亏、逐日事实盈亏趋势”。
- 基础套保和战略套保显示“业务口径、期间业务归属盈亏、逐日业务归属盈亏趋势”。
- 数据质量区显示真实未归属数量、快照日期和异常状态。
- 左侧菜单、页面标题和面包屑统一显示“持仓与交易明细”。
- “持仓与交易明细”副标题为“查询和核验全部成交、平仓及持仓事实”。

### 13.5 性能与真实页面验收

- 在当前 Staging 数据规模和非冷启动状态下，四种时间筛选及三种统计范围切换满足第 11 节性能要求。
- 使用浏览器记录至少一次“全部、基础套保、战略套保”和四种时间维度的真实响应耗时。
- 浏览器控制台无相关 error 或 warning。
- 页面在常用桌面宽度下无筛选控件裁切、重叠或横向失控。
- 自动化测试必须验证真实交互行为和接口参数，不得只检查源码中是否存在按钮、变量或文字。

### 13.6 回归验收

- “持仓与交易明细”的当前持仓、平仓记录、全部交易、分页、筛选、详情和业务归属操作无回归。
- 上海钧能台账、期权台账、结算单导入、事实版本、业务重配和权限控制无回归。
- 原始事实数量、事实盈亏和业务归属结果在本次修改前后保持一致；本次只改变查询、汇总和显示，不改写既有业务数据。

## 15. D/T/R/C 评估

- **D3**：修改集中在交易管理一个业务模块，但同时涉及事实/业务统计语义、汇总查询、前端筛选状态和真实页面显示。未达到 D4，因为不改变其它业务模块、系统级权限架构、外部数据源或跨模块业务流程。
- **T3**：需要覆盖交易事实、业务归属、汇总 API、前端筛选、并发行为和真实 Staging 页面的一体化验收。未达到 T4，因为验收链路始终限制在交易管理模块内，不包含跨模块下游业务动作。
- **R2**：Staging 内可回滚，但统计口径错误会直接影响管理判断，因此必须进行真实数据回归。Production、正式数据库和生产发布不在本次授权范围内；进入 Production 时重新评估为 Gate B。
- **C1**：由一个主 Agent 完成，范围明确，不使用多 Agent，不需要新增外部服务。

保持不变的区域：原始事实、结算单导入、业务归属存储、业务重配审计、权限编码、其它业务模块、Production 环境和正式数据。
