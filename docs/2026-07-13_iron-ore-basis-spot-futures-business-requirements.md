- 不操作 main、Production 或生产数据库。

## 4. 数据范围与计算口径

### 4.1 港口与顺序

日照港、青岛港、岚山港、连云港、江阴港、太仓港、京唐港、曹妃甸港。展示页港口页签按此顺序排列，默认日照港。

### 4.2 品种

卡拉拉精粉、卡拉加斯粉、乌克兰精粉、昆巴粉、BRBF、纽曼粉、PB粉、IOC6、麦克粉、罗伊山粉、金布巴粉、SP10粉、FMG混合粉、杨迪粉、超特粉。

### 4.3 计算规则

```text
干吨现货价 = 湿吨现货价 / (1 - H2O)
质量升贴水 = Fe + SiO2 + Al2O3 + P + S 分项升贴水
标准化现货价 = 干吨现货价 - 质量升贴水 - 品牌升贴水
基差 = 标准化现货价 - I0主力连续收盘价
```

- 规则使用 I2312 / F-DCE I004-2021，Fe 调整参数 X=1.5；
- EBC 只取目标品种的湿吨价格；同品种多规格取高品位报价；
- 质量元素和水分使用对应年度 Excel 参数，不使用 EBC 名称中的 Fe；
- 乌克兰精粉固定 Fe65.54%、SiO2 7.5%、Al2O3 0.11%、P 0.07%、S 0.03%、H2O 9%；
- S 为空按 0；PB粉 P 原值 0.11 按 0.11%换算；卡拉拉精粉使用保证值；
- 昆巴粉使用 EBC 南非粉63%Fe湿吨价和 Excel 昆巴粉参数；
- 杨迪粉使用 EBC 56.8%Fe湿吨价和 Excel 58.3%参数；
- SP10粉在 2025、2026 年覆盖为 SiO2 5.7%、Al2O3 3.2%，2024 年保留年度原值；
- 期货使用 I0 主力连续合约当日收盘价；只有同日现货和盘面同时存在才计算；
- 港口或品种无数据时不生成记录，不补零、不前向填充。

## 5. 数据模型

### 5.1 期现结果表

供页面查询，至少保存：技术主键、业务唯一键、日期、周次、年份、港口、品种、湿吨现货价、质量升贴水、品牌升贴水、标准化现货价、I0主力连续收盘价、基差、数据状态、规则版本、参数版本、来源工作簿摘要和时间戳。

业务唯一键：

```text
日期 + 港口 + 品种 + 规则版本 + 参数版本
```

### 5.2 期现计算明细表

按业务唯一键一一关联结果表，保存工作簿“计算明细”的全部 39 个业务字段，包括 EBC 指标、年度参数、五项升贴水分项、干吨价、标准化现货价、期货序列、规则和参数来源。该表只供后台数据库查询，不单独建设页面。

### 5.3 导入规则

- 工作簿必须包含“期现数据”和“计算明细”；
- 两表必须各 60,424 行、业务唯一键无重复且逐条一致；
- 导入在同一事务中完成，任一校验或写入失败则整体回滚；
- 重复导入相同业务唯一键执行更新，不产生重复记录；
- 当前确认文件 SHA-256 为 `b813c2897c5631ee5fe09db63bfc005e6433ae6929f7b2aab3a7d57d817f917c`。

## 6. 期现数据管理

- 页面位于左侧导航“数据可视化管理 / 数据管理 / 期现数据管理”；
- “数据管理”必须在左侧菜单中展开为“现货数据管理 / 期现数据管理”两个真实三级菜单，禁止使用页面内部页签替代；
- 三级菜单继承父级 `data_visualization_data` 权限，不新增权限编码或数据库迁移；
- 只读，无导入、导出、编辑和删除按钮；
- 筛选：年份、品种、港口，默认全部；
- 年份、品种、港口必须复用现货数据管理相同的复选筛选面板、全选和全不选操作，不使用独立的裸复选框布局；
- 表格：日期、周次、年份、港口、品种、湿吨现货价、质量升贴水、品牌升贴水、主力连续收盘价、基差、数据状态；
- 复用现有数据管理的表格密度、筛选样式和“加载更多”分页逻辑；
- 页面查询不得一次返回全量数据。

## 7. 期现数据展示

- 页面位于左侧导航“数据可视化管理 / 数据展示 / 期现数据展示”；
- “数据展示”必须在左侧菜单中展开为“现货数据展示 / 期现数据展示”两个真实三级菜单，禁止使用页面内部页签替代；
- 三级菜单继承父级 `data_visualization_chart` 权限，不新增权限编码或数据库迁移；

### 7.1 最优仓单测算

- 只出现在数据展示页顶部，与筛选项和港口页签完全独立；
- 仅看当前自然年，取系统中最近一个存在有效基差记录的日期；
- 在该日全港口、全品种有效记录中选择基差最小值；
- 并列时依次按标准化现货价、湿吨现货价、港口、品种升序确定；
- 展示：数据截至日期、品种、港口、湿吨现货价、质量升贴水、品牌升贴水、标准化现货价、I0主力连续收盘价、基差；
- 不展示 EBC 指标名/编码、规则版本、参数版本，不提供点击和明细弹窗；
- 当前线下验收基准：2026-07-10，昆巴粉，京唐港，湿吨价746，质量升贴水28，品牌升贴水0，标准化现货价739.4897119341564，I0收盘价751.5，基差-12.01028806584361。

### 7.2 筛选与港口页签

- 图表筛选仅保留年份、品种，默认全部；
- 年份、品种必须复用现货数据展示相同的复选筛选面板、全选和全不选操作；
- 港口改为页签，默认日照港；
- 只请求当前页签港口的数据，不预取其他港口；
- 最优仓单测算不随筛选和页签变化。

### 7.3 基差图表

- 按品种生成小多图；
- 固定港口下，以年份区分折线，最多 3 条年度曲线；
- 每个实际交易日一个点，横轴必须完整显示 1 月至 12 月；
- 年份图例只在图表区右上角统一显示一次，不得在每个品种小图内重复；
- 鼠标停留在折线数据附近时，以 HTML 提示框显示最近一个真实数据点的日期、年份、品种、港口和基差，不插值、不生成不存在的日期；
- 点击任一年度折线后，同年度折线在全部品种小图中高亮、其他年度淡化；再次点击同年度取消高亮；
- 纵轴允许负数并明确绘制 0 轴；
- 缺失日期不补点、不补零；
- 复用当前数据展示的 Canvas、颜色、图例、响应式和交互风格。

## 8. 权限与异常

- 期现数据管理继承 `data_visualization.data:view`；
- 期现数据展示继承 `data_visualization.display:view`；
- 导入器只作为后台命令，不暴露网页接口；
- 最优仓单加载失败不得阻断图表；图表加载失败只影响图表区域；
- 后续 API 数据未达到 T-1 时必须显示实际数据截至日期，不伪装为最新。

## 9. 验收标准

- SQLite 与 PostgreSQL 建表均幂等；Supabase 公共表启用 RLS，不向 Data API 公共角色授权；
- 本地导入结果表 60,424 行、明细表 60,424 行，重复导入仍保持相同行数；
- 左侧菜单存在真实三级父子结构，页面内部不存在现货/期现切换页签，面包屑显示三级路径；
- 管理页筛选、分页和字段正确，期现筛选与现货筛选使用相同组件及全选/全不选交互；
- 展示页默认日照港，只请求当前港口；年份/品种筛选只影响图表并与现货筛选使用相同组件；
- 最优仓单固定返回上述 2026-07-10 验收记录；
- 基差图表包含负值和清晰 0 轴，横轴完整显示 1 月至 12 月，年份图例只出现一次；
- 点击年度折线能够高亮并再次点击取消，悬停能够显示真实日期和基差；
- 原现货数据管理和现货数据展示无回归；
- staging 页面无控制台应用错误，桌面页面无裁切或横向失控；
- 通过后停在 Gate B，不自动发布生产。
