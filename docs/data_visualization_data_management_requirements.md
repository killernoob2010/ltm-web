# 数据可视化管理 - 数据管理模块需求调整

## 目标

数据管理模块不再直接解析原始 Mysteel Excel，也不再围绕卡粉单品种 MVP 设计。后续数据管理只消费数据整合模块产出的标准整合结果，并负责筛选、排列、查看和导入整合 Excel。页面内双击修改数据的旧功能取消，数据修正回到源 Excel / 数据整合层处理。

## 标准数据源

唯一标准输入为数据整合模块生成的整合结果表 / 整合 Excel。

整合 Excel 固定包含 5 个 sheet：

- `整合明细`：唯一导入数据源。
- `发运`：按 `数据类型=发运/到港` 从整合明细拆分，仅用于查看。
- `库存`：按 `数据类型=库存` 从整合明细拆分，仅用于查看。
- `表需`：按 `数据类型=表需` 从整合明细拆分，仅用于查看。
- `批次信息`：来源文件、批次和数量统计。

数据管理模块导入整合 Excel 时只读取 `整合明细`，不得重复读取 `发运/库存/表需` 三个拆分 sheet，避免重复入库。

## 整合明细字段

`整合明细` 必须包含以下字段：

```text
统计周一
统计周日
业务年份
业务周次
周次标签
展示日期
数据类型
来源/国家
品种
种类
主流/非主流
数值
单位
来源文件
来源Sheet
来源区域
是否参与表需
校验状态
备注
```

其中：

- `统计周一/统计周日/业务年份/业务周次/周次标签` 由数据整合模块生成。
- `数据类型` 允许值：`库存`、`发运/到港`、`表需`。
- `种类` 允许值按当前数据源动态出现，当前包含：`粉矿`、`块矿`、`球团`、`精粉`、`全品种`。
- `主流/非主流` 允许值：`主流`、`非主流`。
- `是否参与表需` 是审计字段，不作为默认筛选条件。
- `单位` 当前统一为 `万吨`。

## 模块职责边界

数据整合模块负责：

- 原始 Mysteel Excel 解析。
- 品种映射、种类映射、主流/非主流判断。
- 日期归周、业务周次生成。
- 澳洲/巴西表需计算。
- 生成标准整合结果和可下载 Excel。

数据管理模块负责：

- 读取标准整合结果。
- 按筛选条件展示宽表。
- 支持导入已整合 Excel。

数据管理模块不得：

- 重新识别原始 Mysteel Excel。
- 重新推导业务周次。
- 重新计算表需。
- 写死 `卡粉` 或任何固定品种列表。
- 通过双击单元格或其他隐式页面交互直接修改已入库数据。

## 展示方式

数据管理保留三个页签：

```text
库存 | 发运 | 表需
```

每个页签根据 `数据类型` 过滤整合结果。

表格建议继续使用宽表：

```text
展示日期 | 周次标签 | 品种1 | 品种2 | 品种3 | ...
```

品种列必须从当前整合结果动态生成。若同一品种出现在多个种类中，列名使用：

```text
品种（种类）
```

## 筛选条件

数据管理需要支持以下筛选：

- `业务年份`
- `业务周次 / 周次标签`
- `品种`
- `种类`
- `来源/国家`
- `主流/非主流`

筛选选项全部从当前整合结果动态生成。

## 数据修正口径

当前阶段不在数据管理页提供人工修正入口。发现数据错误时，应先修正源 Excel 或数据整合结果，再重新导入标准整合 Excel。

后续如确实需要线上修正，应另做显式功能，不使用双击单元格：

- 明确的“修正”按钮或详情入口。
- 原值、新值、修改人、修改时间、修正原因。
- 可审计、可回滚的修正记录。
- 与导入权限分离的单独权限控制。

## 需要同步修复的旧问题

- 移除卡粉硬编码：旧 `PRODUCT = "卡粉"` 和前端只显示卡粉的逻辑不能继续用于数据管理。
- 移除数据管理阶段的周次二次计算：周次由整合结果提供。
- 移除旧统一 `t-2` 表需口径：表需只读取整合结果，不在数据管理重算。
- 数据管理和图表展示应优先读取整合结果，不依赖旧 `dv_data_points` 单品种结构。
- 导入已整合 Excel 时需要明确只读取 `整合明细`。
- 移除旧页面双击改单元格和 `PUT /api/data-visualization/value` 手工改值入口。

## 验收标准

- 数据管理页可展示当前整合结果中的全量品种。
- 品种、种类、来源/国家、主流/非主流筛选可用。
- 表格周次直接来自 `周次标签`，不再由前端或数据管理接口二次计算。
- 库存、发运、表需页签的数据量与整合结果一致。
- 数据管理表格为只读展示，不支持双击修改单元格。
