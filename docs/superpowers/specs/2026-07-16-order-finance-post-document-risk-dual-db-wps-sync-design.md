# 订单融资交单后风险与双库 WPS 同步设计

## 1. 目标

本次调整解决两个问题：

1. 交单后的总体风险只反映未回款融资，不再被已经完成的装船、交单节点或其他旧预警抬高。
2. Staging 与 Production 继续使用两个隔离数据库，但分别从同一份 WPS 台账自动同步事实数据，并保持相同的调度、解析、校验和更新机制。

本设计定向替代 `2026-07-15-order-finance-wps-sync-and-workflow-design.md` 中“4.3 回款风险”“4.4 其他风险”和双环境授权未明确的部分；未被本设计修改的阶段、页面和同步规则继续有效。

## 2. 已确认风险口径

### 2.1 交单前

没有交单日期时，继续沿用现有装船、交单、融资到期、WPS 有效预警和人工跟进提醒规则。本次不调整交单前的风险口径。

### 2.2 交单后

只要项次存在 WPS 交单日期，总体风险只评估该项次下尚未回款的融资：

- 未回款且有效融资到期日在今天之后：中风险。
- 未回款且有效融资到期日就是今天：高风险。
- 未回款且有效融资到期日在今天之前：高风险。
- 未回款且有效融资到期日缺失：中风险，同时产生“融资到期日缺失”的数据异常。
- 所有融资均已回款但尚未结案：低风险，阶段为`已回款待结案`。
- WPS 明确结案、已完成或已结算：显示`已完成`，退出高、中、低风险统计。

交单后，装船风险、交单风险、WPS 中的装船/交单/其他业务预警和人工跟进提醒可以继续展示原始信息，但不得再抬高总体风险，也不得仅因这些已忽略因素把项次加入本周重点。回款类 WPS 提示只作为辅助文案，最终等级仍由未回款状态和有效融资到期日统一计算。

### 2.3 多笔融资

- 已回款融资不再参与当前风险计算。
- 任意一笔未回款融资在当天到期或已经逾期，项次总体为高风险。
- 没有当天到期或逾期融资，但仍存在未回款融资，项次总体为中风险。
- 所有融资均已回款时，项次总体为低风险，等待结案。
- 多笔融资中缺少到期日的未回款记录产生数据异常；如果另有当天到期或逾期融资，总体仍为高风险，否则为中风险。

### 2.4 日期边界

日期按北京时间自然日比较，不按同步发生时刻比较。有效融资到期日继续沿用现有字段优先级；本次不改动原到期日、新到期日和展期天数的计算口径。

## 3. 本周重点与页面表现

- 已交单项次只有总体风险为高风险时进入`本周重点`。
- 已交单但尚未到期的中风险项次不自动进入`本周重点`。
- 已交单项次的风险着色只落在回款状态或融资到期日相关字段，不再把装船状态、交单状态或备注提醒标红。
- 到期日缺失时显示中风险，并在数据异常中明确显示“融资到期日缺失”；不得显示为“已逾期”。
- 已回款待结案显示低风险；已完成继续使用独立完成样式。

## 4. 双库 WPS 自动同步

### 4.1 环境隔离

- Staging Render 只写入 Supabase `LTM WEB STAGING`。
- Production Render 只写入 Supabase `LTM WEB`。
- 两个环境不得共用数据库连接，不执行数据库间自动复制，也不把 Staging 测试记录写入 Production。

### 4.2 同源和同机制

两个环境分别读取同一份 WPS 企业共享台账，并使用同一套代码路径：

- 每天北京时间 09:00 和 17:00 检查一次，包括周末和节假日。
- 读取相同的 drive、file 和`订单 / 额度 / 预警`三个页签。
- 使用相同的业务键、字段标准化、完整性校验、事实字段比较、事务写入和记录归档规则。
- 分别记录本环境的最近成功时间、变化条数、源文件版本和最近尝试时点。
- 同一源文件版本重复同步时变化条数为 0。

允许两个环境的实际完成时间相差几分钟；一致性的判断标准是最近成功的源文件版本和 WPS 事实字段一致，而不是时间戳完全相同。

### 4.3 独立授权

- Staging 和 Production 分别持有独立的 WPS 用户授权 refresh token。
- 两个环境可以读取同一文件，但不得复制同一个可轮换 refresh token 作为长期配置。
- access token 和轮换后的 refresh token 只在各自环境及各自同步状态中使用，不跨库同步。
- 任一环境授权失效只影响该环境，另一环境继续正常同步。

### 4.4 字段覆盖边界

WPS 自动同步只更新事实字段。以下管理字段在两个数据库中独立保存，不要求一致，也不得被 WPS 覆盖：

- 下一步。
- 回款要求及其状态。
- 管理备注。
- 计划跟进日。
- 人工装船确认及其审计信息。

因此，“双库同步一致”指 WPS 源版本和事实字段一致，不指用户、权限、操作日志、测试记录或人工管理字段完全一致。

### 4.5 失败保护

以下任一情况发生时，本次同步不得写入或归档任何订单融资事实数据，也不得覆盖上次成功状态：

- WPS 授权或下载失败。
- 必需页签缺失。
- 工作簿无法解析。
- `订单`页签为空。
- 关键字段或业务键校验失败。
- 记录数量或业务键集合出现不能解释的异常缩减。
- 数据库事务失败。

业务键集合比当前有效 WPS 事实记录减少时，按以下两次确认机制处理：

1. 第一次发现减少，只记录待确认的源文件版本、业务键集合指纹和记录数量，不写入或归档数据，也不覆盖上次成功状态。
2. 后续同步仍得到相同源文件版本和相同业务键集合指纹时，确认该缩减来自稳定源文件，允许按正常事务归档缺失业务键。
3. 后续同步的源文件版本或业务键集合发生变化时，用新的候选替换旧候选并重新等待一次确认。
4. 手动上传不受两次确认限制，继续按现有人工确认边界执行。

该机制不自动判断 WPS 删除是否正确，只防止瞬时残缺文件在两个数据库中立即造成错误归档。服务端记录脱敏状态，保留上次成功数据；合法删除最多延迟一个同步周期生效。

## 5. 一致性检查

每个环境独立同步成功后，至少能够确认：

- 最近成功源文件版本。
- 有效业务键数量。
- 新增、更新、归档和总变化数量。
- 数据异常数量。

Staging 验收时对两边最近成功的源文件版本和事实字段业务指纹做只读比较。比较不包含数据库 ID、创建/更新时间、用户管理字段、权限、会话和操作日志。

本次不建设新的跨环境同步页面或持续对账服务；一致性比较作为发布前和故障排查时的只读验收步骤。

## 6. 验收标准

### 6.1 风险规则

- 已交单、未回款、到期日在明天或更晚：中风险。
- 已交单、未回款、到期日为今天：高风险。
- 已交单、未回款、到期日已过：高风险。
- 已交单、未回款、到期日缺失：中风险并产生数据异常，页面不显示逾期。
- 已交单、多笔融资部分回款：只评估未回款融资并取最高等级。
- 已交单、全部回款但未结案：低风险、`已回款待结案`。
- 已完成：退出风险统计。
- 已交单后，装船/交单预警、其他 WPS 预警和人工提醒不能改变总体风险或字段风险着色。

### 6.2 双库同步

- Staging 与 Production 分别使用独立数据库和独立 refresh token读取同一 WPS 文件。
- 两边使用相同的 09:00、17:00 调度和相同解析、校验、事务更新机制。
- 同一 WPS 源版本同步成功后，两边事实字段业务指纹一致。
- 管理字段在任一环境修改后，下一次 WPS 同步仍保留本环境的值。
- 缺页、空表、异常缩减、解析失败或数据库失败时，不发生部分覆盖或错误归档。
- 首次出现业务键缩减时不写库；下一次源版本和业务键集合均相同后才执行归档。
- 一个环境同步失败时，另一个环境不受影响。

## 7. 影响范围与非目标

影响范围仅包括订单融资模块的风险计算、字段风险着色、本周重点判断、WPS 自动同步保护及相关测试。

本次不做：

- 不合并 Staging 与 Production 数据库。
- 不复制用户、权限、会话、日志、测试记录或管理字段。
- 不新增跨环境实时复制服务。
- 不调整交单前的装船和交单规则。
- 不修改融资资金监控的额度计算口径。
- 不新增 WPS 写入、修改、上传或删除权限。
- 不在未完成 Staging 验收和 Production Gate B 确认前修改正式配置、正式数据库或正式部署。
