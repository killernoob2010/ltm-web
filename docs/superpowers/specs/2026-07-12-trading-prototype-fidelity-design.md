# 交易管理原型高保真还原设计

## 基准与强制约束

- 唯一页面原型基准：`/Users/wangjingze/Documents/交易台账自动化处理/personal_workbench/trading_static/`。
- 固定原型提交：`c1de82b`。实施前和验收时均以该提交中的 `index.html`、`styles.css`、`app.js` 为逐项对照来源。
- 左侧导航保持轻量化交易管理系统现有统一导航，不复制原型的独立左侧栏。
- 左侧导航右侧的交易管理工作区，必须完整复用原型的信息结构、布局比例、视觉层级、组件形式、字段顺序和交互方式。
- 原型是实现基准，不是仅供参考的视觉草图。除后续经用户明确确认的业务调整外，不允许自行替换布局、删减组件或用系统通用组件重新设计。

## 工作区外壳

进入任一交易管理二级菜单时：

- 保留系统现有左侧统一导航及其折叠、权限和激活状态。
- 右侧不再使用当前交易管理的通用工具栏、指标卡和简化表格布局。
- 右侧使用原型工作区标题栏：页面标题、副标题、账户筛选、日期筛选和“数据说明”。
- 原型中的“只读数据分析”安全语义保留在右侧工作区，不增加任何真实交易操作。
- 原型 CSS 以交易管理命名空间隔离，避免覆盖旧台账和系统其它页面。

## 五个页面逐页还原

### 总览

- 保留原型的日、月、季、自定义区间选择条。
- 使用横向汇总带，不改为独立大卡片。
- 保留成交笔数/手数、事实平仓盈亏、手续费、期末持仓、保证金等原型指标层级。
- 保留“逐日平仓盈亏趋势”“数据质量”“业务归属分布”“活跃合约”区块及原型双栏比例。
- 业务尚未完成或当前接口无法计算的值在原位置显示“待计算”或“待核验”，不得删除对应组件。

### 持仓与交易

- 标签顺序固定为“当前持仓、平仓记录、全部交易”。
- 保留原型的搜索、资产类型、方向、开平、归类状态、日期范围等筛选组件及筛选汇总带。
- 保留 20/50/100 分页、当前页选择、全部筛选结果选择和清空选择。
- 业务归属继续作用于完整成交身份，不允许按手数拆分。
- 三表导入入口只放在本页面；使用原型右侧抽屉的视觉与层级，不在其它四个页面重复显示。
- 导入抽屉使用真实账户和三个文件控件，并叠加已经确认的预检、确认覆盖、处理中反馈和失败恢复规则。

### 上海钧能台账

- 使用原型 `ledger-hero`、摘要区、筛选区、统一标签页、汇总带、表格和分页结构。
- 标签顺序固定为“当前持仓、平仓记录、全部交易”。
- 正式汇总只纳入已归属上海钧能的数据；RB、HC 未归类数据继续作为候选提示，不混入正式汇总。
- 业务开平调整沿用原型详情抽屉，不另做居中弹窗；在抽屉内增加候选、影响预览、确认和恢复默认。
- 事实盈亏与业务盈亏均在原型相应明细/核验位置展示。

### 期权台账

- 使用原型统一标签页、筛选汇总、表格和分页结构。
- 默认展示全部期权，标签顺序固定为“当前持仓、平仓记录、全部交易”。
- 保留标的、看涨/看跌、行权价、权利金及 Delta、Gamma、Theta、Vega 的原型字段位置。
- 尚未确认的浮动盈亏和风险指标在原型位置显示“待计算”，不删除列或改成页面外说明。

### 汇总与导出

- 视觉结构保留原型的“统一输出”“输出前检查”和“最近输出版本”。
- 根据当前已确认范围，将所有真实导出按钮置为不可执行状态，并明确显示功能暂未开放。
- 不使用当前单一居中空状态替代原型完整结构。

## 共用组件

下列组件直接以原型为基准实现，并只做数据接口适配：

- 工作区标题栏与筛选控件；
- `summary-band` / `filter-summary`；
- `content-grid` / `ledger-hero`；
- `panel`、`panel-header`、`section-header`；
- 标签页、筛选器、表格、分页；
- 状态标签、数据质量行、业务分布行；
- 右侧详情/操作抽屉、遮罩和 toast；
- 响应式断点和窄屏单列行为。

组件可增加交易管理命名空间以避免 CSS 冲突，但不得以命名空间调整为理由改变原型视觉结果。

## 已确认的原型差异

只有以下差异允许覆盖原型：

1. 左侧使用现有系统统一导航，不使用原型独立左侧栏。
2. 后端使用正式事实层、业务归属层和业务开平关系层，不使用浏览器内存模拟数据。
3. 三表导入只在“持仓与交易”显示，支持真实预检与确认覆盖。
4. 导入过程中必须有抽屉内 loading、按钮禁用、成功/失败状态。
5. 业务类型固定为基础套保、战略套保；业务主体和策略按已确认配置执行。
6. 浮动盈亏和期权风险指标暂显示“待计算”。
7. 汇总与导出保留原型结构但本期不执行真实导出。
8. 业务开平关系可人工调整，并同步业务持仓和平仓继承；事实层不变。

未列入上述清单的页面差异一律视为实现缺陷。

## 数据适配原则

- 原型组件读取正式 API 返回的数据，不复制第二套前端业务口径。
- 若正式 API 暂缺原型某组件所需聚合字段，优先扩展统一查询响应，不删除原型组件。
- 明细与汇总必须使用相同筛选条件。
- 无数据时保留原型组件结构并显示空状态，不切换到另一套页面。

## 验收方法

- 以 `c1de82b` 原型和正式页面做逐页并排截图比较。
- 对总览、持仓与交易、上海钧能、期权、汇总与导出分别核对区块顺序、栅格比例、字段顺序、标签顺序、筛选器、分页、抽屉和空状态。
- 增加前端契约测试，直接校验原型关键组件、可见文案、标签顺序和允许差异清单。
- 浏览器验证左侧统一导航未改变，五个右侧工作区符合原型，控制台无错误。
- 未经确认出现的缺失区块、通用大卡片替代、居中弹窗替代右侧抽屉、标签或字段重排，均判定验收失败。
