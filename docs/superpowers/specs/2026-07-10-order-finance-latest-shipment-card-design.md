# 订单融资日期展示与风险口径修正设计

## 目标

订单融资卡片展示 Excel `订单` 页签的 `最迟装船日`，帮助业务人员在日期前一周主动向工厂确认装船进度；同时修正展期、结案还款及时性、多笔融资和风险分级口径。

## 范围

- 卡片字段名称由 `提单日` 改为 `最迟装船日`。
- 后端从 `订单.最迟装船日` 写入现有 `latest_shipment_date` 字段，并在项次聚合结果中返回该日期。
- 卡片显示 `日期 / 距今天数`：超过 7 天为普通样式，0 至 7 天为黄色提示，已过期为红色提示。
- 最迟装船日为空时显示 `待 Excel 补充`。
- 卡片 `还款情况` 改为 `还款日`；存续业务为空时显示 `待还款`，已结案业务为空时显示 `未提供`。
- 展开后的融资明细删除当前没有可靠数据来源的 `提单日` 列，保留 `交单日` 和 `还款日`。
- `缺提单/交单/还款` 文案改为 `缺最迟装船/交单/还款`，计算时使用最迟装船日，不使用提单日。
- 已放款待装船的下一步提示改为联系工厂确认装船进度，不再提示确认提单进度。
- 不新增顶部指标、筛选按钮、自动通知或后台定时提醒。

## 有效融资到期日

- 风险和还款及时性优先使用 Excel `新到期日`。
- `新到期日` 为空且 `展期天数` 大于 0 时，使用 `原到期日 + 展期天数`。
- 没有展期时使用原到期日。
- 已结案订单不再用有效到期日与今天比较；改为使用 `真实还款日 - 有效到期日`，展示 `提前 N 天还款`、`按期还款` 或 `逾期 N 天还款`。

## 风险分级

- 已结案：显示 `已完成`，不参与高、中、低风险和数据异常数量；2025 年已结案数据同样忽略异常。
- 高风险：存续业务尚未还款，且有效融资到期日已经逾期或不足 7 天；或者存续业务存在真正严重的数据异常。
- 中风险：存续业务有效融资到期日在 8 至 30 天内；或者已经填写还款日但 Excel 仍标记存续，需要确认结案。
- 低风险：存续业务有效融资到期日在 30 天以后，且没有严重数据异常。
- 最迟装船日不足 7 天只控制卡片黄色提示，不直接改变融资风险等级。
- 同一项次存在多笔融资只显示 `多笔融资` 标记，不作为数据异常，也不改变风险等级。

## 不改变的业务逻辑

- `最迟装船日` 不等于 `提单日`，两者不互相填充。
- 页面布局、卡片数量、字段总数和融资资金监控页面保持不变。

## 数据与边界情况

- 同一项次存在多笔融资时，项次卡片展示该组中最早的非空最迟装船日，以便按最紧迫日期跟进；每笔融资仍独立使用自己的有效融资到期日。
- 已结案业务仍展示其最迟装船日；没有记录时显示 `未提供`，避免将历史缺失误解为待办。
- 无法识别的日期继续进入数据异常，不作为有效倒计时日期。

## 验证

- 后端测试：确认 Excel 最迟装船日进入记录与聚合结果；同项次多行取最早日期；新到期日和展期天数生成有效到期日；结案业务不参与风险和异常；多笔融资不改变风险；真实还款日与有效到期日生成及时性结果。
- 前端测试：确认卡片名称、日期倒计时、7 天内黄色、逾期红色、空值文案、还款日字段、结案及时性，以及展开明细没有提单日列。
- 真实台账测试：使用最新 `YOLANDA和香港建龙出口钢材信用证台账.xlsx` 验证存续订单日期。
- Staging 浏览器测试：确认页面字段、倒计时、颜色、展开明细和控制台状态。
