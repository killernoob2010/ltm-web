# 订单融资状态与摘要显示返工设计

## 目标

在不改变订单融资风险评价规则的前提下，统一装船、交单和多笔融资摘要的前端文案，并将“数据异常”收敛为纯数据质量问题。

## 业务规则

- WPS“预警”页签仍参与订单风险评价，但不计入“数据异常数”或导入完成提示中的异常数。
- 数据异常只统计系统识别出的数据质量问题，例如日期无法识别、日期年份异常、融资到期日早于放款日期、存续/结案状态为空。
- 有交单日时，后台继续自动认定已经装船；前端装船状态固定显示“已根据交单日认定装船”，不附交单日期。
- 有交单日时，交单状态显示“已交单 / YYYY-MM-DD”；无交单日时显示“待交单”。
- 多笔融资摘要显示为“银行 / 金额（N笔）”；单笔融资摘要保持“银行 / 金额”。摘要保持单行，空间不足时使用省略号。

## 不变范围

- 不修改风险等级、风险指标、本周重点、业务阶段和下一步动作的计算。
- 不修改 WPS 下载、同步频率、数据库结构或已有业务数据。
- 不进入 Gate B，不发布 Production。

## 验收

- 自动测试证明 `excel_alert` 不计入数据异常，但仍保留在风险输入中；数据质量告警仍计入异常。
- 自动测试证明三项前端文案符合上述格式。
- Staging 使用真实 `H-2026-3` 检查装船、交单和多笔融资摘要的实际单行显示。

## D/T/R/C

- D2：订单融资单一模块内的统计口径和显示返工，不涉及其他业务模块。
- T2：覆盖模块后端统计与前端显示，并在真实 Staging 页面验收目标卡片；不需要跨模块集成测试。
- R1：无数据库迁移和数据写入，代码可直接回滚。
- C1：单 Agent、小范围修改。
