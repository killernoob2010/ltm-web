# 数据可视化管理 - 数据管理模块需求调整

## 目标

数据管理模块不再直接解析原始 Mysteel Excel，也不再围绕卡粉单品种 MVP 设计。后续数据管理只消费数据整合模块产出的标准整合结果，并负责筛选、排列、查看和人工修正。

## 标准数据源

唯一标准输入为数据整合模块生成的整合结果表 / 整合 Excel。

整合 Excel 固定包含 5 个 sheet：

- `整合明细`：唯一导入数据源。
- `发运`：按 `数据类型=发运/到港` 从整合明细拆分，仅用于查看。
- `库存`：按 `数据类型=库存` 从整合明细拆分，仅用于查看。
- `表需`：按 `数据类型=表需` 从整合明细拆分，仅用于查看。
- `批次信息`：来源文件、批次和数量统计。

数据管理模块导入整合 Excel 时只读取 `整合明细`，不得重复读取 `发运/库存/表需` 三个拆分 sheet，避免重复入库。

## 整合明细字段

`整合明细` 必须包含以下字段：

```text
统计周一
统计周日
业务年份
业务周次
周次标签
展示日期
数据类型
来源/国家
品种
种类
主流/非主流
数值
单位
来源文件
来源Sheet
来源区域
是否参与表需
校验状态
备注
```

其中：

- `统计周一/统计周日/业务年份/业务周次/周次标签` 由数据整合模块生成。
- `数据类型` 允许值：`库存`、`发运/到港`、`表需`。
- `种类` 允许值按当前数据源动态出现，当前包含：`粉矿`、`块矿`、`球团`、`精粉`、`全品种`。
- `主流/非主流` 允许值：`主流`、`非主流`。
- `是否参与表需` 是审计字段，不作为默认筛选条件。
- `单位` 当前统一为 `万吨`。

## 模块职责边界

数据整合模块负责：

- 原始 Mysteel Excel 解析。
- 品种映射、种类映射、主流/非主流判断。
- 日期归周、业务周次生成。
- 澳洲/巴西表需计算。
- 生成标准整合结果和可下载 Excel。

数据管理模块负责：

- 读取标准整合结果。
- 按筛选条件展示宽表。
- 支持人工修正和修正记录。
- 支持导入已整合 Excel。

数据管理模块不得：

- 重新识别原始 Mysteel Excel。
- 重新推导业务周次。
- 重新计算表需。
- 写死 `卡粉` 或任何固定品种列表。

## 展示方式

数据管理保留三个页签：

```text
库存 | 发运 | 表需
```

每个页签根据 `数据类型` 过滤整合结果。

表格建议继续使用宽表：

```text
展示日期 | 周次标签 | 品种1 | 品种2 | 品种3 | ...
```

品种列必须从当前整合结果动态生成。若同一品种出现在多个种类中，列名使用：

```text
品种（种类）
```

## 筛选条件

数据管理需要支持以下筛选：

- `业务年份`
- `业务周次 / 周次标签`
- `品种`
- `种类`
- `来源/国家`
- `主流/非主流`

筛选选项全部从当前整合结果动态生成。

## 人工修正

人工修正不应覆盖原始整合值，建议使用覆盖层：

```text
原始整合值
人工修正值
当前展示值
是否人工修正
最近修改人
最近修改时间
修正备注
```

当前展示值规则：

```text
如果存在人工修正值，显示人工修正值。
否则显示原始整合值。
```

重新导入整合 Excel 时，不得静默覆盖已有人工修正。需要预检提示：

- 新增数据量
- 覆盖原始整合值数量
- 人工修正保护数量
- 缺失/异常数量

## 需要同步修复的旧问题

- 移除卡粉硬编码：旧 `PRODUCT = "卡粉"` 和前端只显示卡粉的逻辑不能继续用于数据管理。
- 移除数据管理阶段的周次二次计算：周次由整合结果提供。
- 移除旧统一 `t-2` 表需口径：表需只读取整合结果，不在数据管理重算。
- 数据管理和图表展示应优先读取整合结果，不依赖旧 `dv_data_points` 单品种结构。
- 导入已整合 Excel 时需要明确只读取 `整合明细`。

## 验收标准

- 数据管理页可展示当前整合结果中的全量品种。
- 品种、种类、来源/国家、主流/非主流筛选可用。
- 表格周次直接来自 `周次标签`，不再由前端或数据管理接口二次计算。
- 库存、发运、表需页签的数据量与整合结果一致。
- 人工修正不会被后续导入静默覆盖。
