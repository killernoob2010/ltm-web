# 订单融资集港状态与风险规则设计

## 目标

在现有订单融资进度中补齐“集港”环节，使页面状态按真实业务顺序流转，并在已集港待装船阶段按最迟装船期自动调整风险。

本次只修改订单融资进度模块，不改变融资资金监控、WPS 事实同步架构、权限模型、其他业务模块或 Production。

## 状态流转

订单融资进度使用以下顺序：

1. `待放款`
2. `已放款待集港`
3. `已集港待装船`
4. `已装船待交单`
5. `已交单待回款`
6. `已回款待结案`
7. `已完成`

状态判断按后续事实优先：已完成 > 已回款 > 已交单 > 已装船 > 已集港 > 已放款 > 待放款。已有交单日继续隐含已经装船，不要求补点前序按钮。

确认集港后仍可确认装船。撤销装船确认时，如果集港确认仍然存在，状态恢复为 `已集港待装船`；撤销集港确认时恢复为 `已放款待集港`。已进入交单、回款或完成阶段后，不显示集港和装船操作按钮。

## 风险规则

### 已放款待集港

沿用当前未装船阶段的风险计算，包括现有 Excel 预警、最迟装船期、融资到期日和人工提醒规则。本次只把阶段名称从 `已放款待装船` 改为 `已放款待集港`，不改变当前业务结果。

### 已集港待装船

- 距离最迟装船期超过 2 天：整体风险为中风险。
- 距离最迟装船期小于或等于 2 天：整体风险为高风险。
- 到达最迟装船日或已经逾期：整体风险为高风险。
- 缺少或无法解析最迟装船期：整体风险为高风险，提示补录日期并确认装船或交单状态。

高风险时下一步显示“确认装船状态或交单状态”；中风险时显示“跟进装船并及时确认状态”。

一旦确认已装船或已有交单日，立即退出集港阶段专用规则，分别回到现有 `已装船待交单` 或 `已交单待回款` 风险逻辑。已完成业务继续显示 `已完成`，不归入高、中、低风险。

## 数据与接口

`order_finance_progress` 增加三个可空人工管理字段：

- `port_confirmed_date`：确认集港日期；
- `port_confirmed_by`：确认人；
- `port_confirmed_at`：操作时间。

字段通过现有幂等数据库初始化逻辑增加，不重建表、不回填推测值。代码回滚时字段可以保留，不影响旧版本读取。

新增 `PATCH /api/order-finance/contracts/{item_no}/port-confirmation`，请求结构与装船确认一致，支持确认和撤销。接口继续使用订单融资编辑权限，并按项次更新该合同下全部融资记录。

集港字段属于环境本地人工管理字段。Excel/WPS 事实快照更新不得覆盖它们，Staging 与 Production 各自保存自己的操作记录；本次不改变快照字段、同步调度或环境隔离。

## 页面交互

- `已放款待集港` 卡片显示“确认已集港”。
- 集港确认使用日期对话框，默认当天，交互与现有装船确认一致。
- `已集港待装船` 卡片显示“确认已装船”和“撤销集港确认”。
- 装船状态栏在集港阶段显示“已确认集港：日期”；确认装船后继续使用现有装船文案。
- 上端筛选栏增加 `已集港待装船`，原 `已放款待装船` 改为 `已放款待集港`。
- 汇总区同步增加 `已集港待装船` 数量，并将原待装船数量改为待集港数量。
- 保持现有卡片结构、搜索、提醒、明细展开和融资资金监控页面不变。

## 自动化验证

1. 有放款、无后续事实时为 `已放款待集港`。
2. 确认集港后为 `已集港待装船`，撤销后恢复待集港。
3. 集港后距离最迟装船期 3 天为中风险；2 天、当天和逾期为高风险。
4. 集港后缺少最迟装船期为高风险，并返回明确下一步。
5. 集港后确认装船进入 `已装船待交单`，按现有装船风险逻辑计算。
6. 已有交单日进入 `已交单待回款`，按现有交单风险逻辑计算。
7. 撤销装船确认且仍有集港确认时恢复 `已集港待装船`。
8. Excel/WPS 重新导入后保留人工集港字段。
9. API 权限、确认、撤销和无效日期错误处理正确。
10. 前端包含确认/撤销交互、新筛选、汇总和静态资源版本，原有装船交互继续存在。

## Staging 验收

在 Render Staging 的真实订单融资进度页面验证：

1. `已放款待集港` 卡片可确认集港，刷新后状态和日期仍然存在。
2. `已集港待装船` 筛选只显示相应业务。
3. 使用最迟装船期边界测试数据核对中、高风险与下一步提示。
4. 集港后确认装船，页面进入现有装船阶段；交单事实存在时进入现有交单阶段。
5. 撤销装船后恢复集港状态，撤销集港后恢复待集港。
6. 页面 URL、标题和静态资源版本正确，控制台无新增错误或警告。

Staging 验收数据必须可精确清理，不操作 Production 数据库、`main`、正式 Render 或正式环境变量。

## 评估与边界

- D2：现有订单融资模块增加一个业务环节和配套交互，没有新增模块或跨模块耦合。
- T2：验证新增状态的完整功能及订单融资模块相关回归，不扩展为跨模块端到端测试。
- R2：新增字段可空、迁移可重复、代码可回滚，业务影响局部且恢复简单。
- C1：单一 Agent 完成。

不因同时涉及数据库、API、前端和测试而机械提升等级；这些都是同一个订单融资状态能力的必要实现层。

## 非目标

- 不调整融资资金监控页面或额度计算。
- 不修改 WPS 文件结构、事实字段、同步调度和双环境快照架构。
- 不改变登录、角色或订单融资权限等级。
- 不改变装船、交单、回款和完成阶段现有风险口径。
- 不新增消息推送、自动写回 Excel/WPS 或其他外部操作。
- 本轮不发布 Production。
