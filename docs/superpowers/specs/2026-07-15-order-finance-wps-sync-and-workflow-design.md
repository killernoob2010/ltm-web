# 订单融资 WPS 自动同步与业务节点重构设计

## 1. 目标

将 WPS 企业共享文档作为订单融资事实数据的权威来源，每天北京时间 09:00 和 17:00 自动读取最新台账，并按“装船—交单—回款—结案”的系统可观测节点重构订单融资阶段、提醒、风险和页面展示。

真实业务顺序为“集港—装船—交单—回款”。当前台账没有可靠的集港字段，系统也不再区分 FCR，因此集港不单独建状态或风险节点。

页面只显示最近一次成功自动同步时间和本次实际变化条数，不增加导入报告、同步历史页面或前端异常清单。异常只进入服务端日志。

## 2. 已确认口径

### 2.1 数据来源

- 线上 WPS 共享文档是 Excel 事实字段的权威来源。
- 自动同步和手动上传沿用“最后一次成功导入生效”的覆盖原则；手动上传后，下一次线上自动同步可再次覆盖 Excel 事实字段。
- 只读取 `订单`、`额度`、`预警` 三个页签，其他页签完全忽略。
- `订单.项次`继续作为订单业务主键；同项次多笔融资按稳定的行序号保留为多条融资记录。
- 重新导入不得覆盖人工备注、跟进日期、人工装船确认等管理字段。

### 2.2 日期语义

- WPS `交单日`是是否已交单的唯一事实依据，不增加人工“确认已交单”按钮。
- 交单是项次级节点；同项次多行中任一行存在交单日，即认定该项次已交单，并在主卡片采用该项次最晚的交单日展示。只有同项次所有行的交单日都被删除，交单状态才回退。
- WPS `还款日`在页面和业务状态中统一解释为“回款日”，表示我方已经收到钱。
- “回款到期日”优先使用`新到期日`；没有新到期日时，使用`原到期日 + 展期天数`。
- 交单截止日为对应融资的`回款到期日 - 15个自然日`。

### 2.3 人工装船确认

- 是否装船继续由卡片上的`确认已装船`按钮人工控制，默认确认日期为当天并允许修改。
- 人工装船确认记录确认日期、确认人和确认时间，并在 WPS 重新同步后保留。
- 如果 WPS 已出现交单日，即使没有人工确认，也自动认定已经装船，并隐藏`确认已装船`按钮。
- 撤销人工装船确认后，如果仍有交单日，系统仍保持已装船；如果没有交单日，则恢复为待装船并重新计算风险。

### 2.4 WPS 关键日期删除

- WPS 后续删除交单日或回款日时，系统事实状态随线上数据回退，并重新计算阶段、提醒和风险。
- 人工装船确认不随 WPS 日期删除而清除。
- 服务端日志记录关键日期由有值变为空的变化，前端不增加异常报告。

## 3. 阶段模型

存续业务使用以下阶段：

1. `待放款`
2. `已放款待装船`
3. `已装船待交单`
4. `已交单待回款`
5. `已回款待结案`
6. `已完成`

判断顺序从后向前，避免较早节点覆盖较晚事实：

- WPS 明确为`结案`：`已完成`。
- 同项次全部融资记录都有回款日：`已回款待结案`。
- 至少一笔融资尚未回款，但 WPS 已有交单日：`已交单待回款`。
- 尚无交单日，但存在人工装船确认：`已装船待交单`。
- 已有放款日期或融资金额：`已放款待装船`。
- 其余：`待放款`。

同项次多笔融资部分回款时，阶段仍为`已交单待回款`，卡片显示“部分回款 X/Y笔”；只有全部融资都有回款日才进入`已回款待结案`。

## 4. 风险与提醒

### 4.1 装船风险

仅在“没有人工装船确认且没有 WPS 交单日”时判断最迟装船日：

- 最迟装船日为空或已经逾期：高风险。
- 最迟装船日在今天至未来10天：中风险。
- 最迟装船日在未来11天及以后：不因装船产生风险。

人工确认装船或 WPS 出现交单日后，最迟装船日及其 Excel 预警不再参与风险评价。

### 4.2 交单风险

对每笔尚未回款的融资单独计算交单截止日，但是否已经交单按项次统一判断：

- WPS 已有交单日：不产生待交单风险。
- WPS 没有交单日，且今天已经到达或超过该笔融资的交单截止日：中风险。
- 多笔融资由最早的未回款交单截止日驱动主卡片提醒和总体风险。
- 已经补录交单日后，即使实际交单日晚于截止日，也解除待交单风险；页面可显示实际交单日，但不保留历史风险。

### 4.3 回款风险

- 尚未回款且距离有效回款到期日8至30天：中风险。
- 尚未回款且距离有效回款到期日不足等于7天或已经逾期：高风险。
- 已回款的融资不再产生该笔回款到期风险。
- 多笔融资的主卡片风险取所有未回款融资中的最高等级。

### 4.4 其他风险

- WPS `预警`页签中的有效高风险继续保留，但已完成业务不参与存续风险。
- 人工跟进日期进入未来10天或已经逾期时，继续产生中风险。
- 总体风险取装船、交单、回款、有效预警和人工提醒中的最高等级。

### 4.5 本周重点

满足任一条件的存续项次进入`本周重点`：

- 总体为高风险。
- 尚未装船，且最迟装船日在今天至未来10天。
- 尚未交单，且已经进入任一未回款融资的15天交单窗口。
- 人工跟进日期在未来10天内或已经逾期。

同一项次即使有多个原因也只计一次，并保留所有入选原因供下一步文案使用。

## 5. 页面调整

### 5.1 顶部同步状态

页面工具栏只显示：

- 上次成功自动同步时间。
- 本次实际变化条数。

自动同步失败不覆盖上一次成功时间和变化条数，只写服务端日志。手动上传仍显示本次上传的即时结果，但不冒充线上自动同步时间。

### 5.2 顶部指标与筛选

保留：

- 未结算业务。
- 存续融资金额。
- 7天内回款到期。
- 30天内回款到期。
- 本周重点。
- 已放款待装船。
- 已装船待交单。
- 已交单待回款。
- 已回款待结案。
- 已完成。
- 数据异常数。

删除`缺最迟装船/交单/还款`指标。阶段筛选同步使用上述新阶段名称。

### 5.3 主卡片

主卡片保留决策所需信息：

- 数量。
- 贷款行／融资金额。
- 装船状态和人工确认日期；有交单日时显示“已根据交单日认定装船”。
- 交单状态；未交单时显示最早交单截止日，已交单时显示 WPS 交单日。
- 回款到期日；展期信息合并在此字段中显示。
- 回款日；部分回款时显示 X/Y 笔，全部回款后显示及时性。
- 下一步。
- 备注／提醒。

删除重复的独立`确认状态`和`展期状态`字段。页面术语统一将`还款日`改为`回款日`，将`融资到期`改为`回款到期日`，将`已还款待结案`改为`已回款待结案`。

### 5.4 展开明细

- 单笔融资只展示主卡片没有的信息：贷款行、利率、原到期日、新到期日、展期天数和来源信息。
- 多笔融资逐笔展示贷款行、金额、放款日、原到期日、有效回款到期日、回款日和状态。
- 项次级交单日只在主卡片展示一次，不在每笔融资行重复。

## 6. 自动同步设计

### 6.1 方案选择

采用现有 Web 服务内置定时任务，复用项目已有后台同步模式：

- 每天北京时间09:00和17:00各执行一次，包括周末和节假日。
- 服务重启后，如果当天最近一个时点已经过去且该时点尚未尝试，则补执行一次；同一时点不得并发或重复执行。
- 不新增付费 Render Cron，不依赖本地 Mac 或 Codex 自动化长期在线。

未采用的方案：

- Render Cron：会增加独立付费服务和环境配置，当前每天两次没有必要。
- 本地 Mac 定时任务：依赖个人电脑在线，不能作为 Web 系统的稳定数据链路。
- WPS 事件订阅：实时性超出业务需要，且增加公网回调、验签和去重复杂度。

### 6.2 WPS 读取

- 使用用户授权访问企业共享文件；应用身份已验证无法直接读取该用户共享文件。
- 定时任务通过用户 refresh token 获取或刷新短期 access token，再按配置的 drive_id 和 file_id读取文件元数据和源文件下载地址。
- 下载源格式 Excel，在内存或临时文件中完成解析和校验；解析完成后删除临时文件。
- 只调用文件元数据和下载类只读接口，不调用上传、修改、分享或删除接口。
- WPS APPID、APPKEY、refresh token、drive_id和file_id只存放在目标环境的秘密配置中，不进入仓库、日志、接口响应或版本记录。
- access token只做短期内存缓存。refresh token失效时同步失败并记录后台日志，需要管理员重新完成一次用户授权。

WPS官方说明用户 access token 有效期为2小时，并返回长期 refresh token；源文件下载接口支持`kso.file.read`用户授权并返回源格式下载地址。因此每天两次同步使用用户授权刷新并下载源文件是可行路径。

官方依据：

- [获取用户 access_token](https://365.kdocs.cn/3rd/open/documents/app-integration-dev/wps365/server/certification-authorization/get-token/get-user-access-token)
- [获取文件下载信息](https://365.kdocs.cn/3rd/open/documents/app-integration-dev/wps365/server/yundoc/file/get-file-download)
- [根据 drive_id 和 file_id 获取文件信息](https://365.kdocs.cn/3rd/open/documents/app-integration-dev/wps365/server/yundoc/file/get-file-info-by-id)

### 6.3 原子导入与变化计数

当前“先归档全部 Excel 行、再全部更新”的实现无法得到真实变化条数，也可能在中途失败时留下不完整状态。新流程改为：

1. 完整下载并解析文件。
2. 校验必需的`订单`页签；`额度`或`预警`缺失时按既有降级规则记录后台异常。
3. 在单个数据库事务中比较当前有效 Excel 记录与新记录。
4. 新业务键插入；事实字段变化的业务键更新；完全一致的不写入；本次已不存在的旧 Excel 业务键归档。
5. 手动新增记录不参与归档；管理字段不参与事实覆盖。
6. 事务成功后更新最近同步状态；任何一步失败都回滚数据和同步状态。

页面显示的“实际变化条数”为：

`新增记录数 + 事实字段更新记录数 + 因线上删除而归档的记录数`

完全相同的文件再次同步显示0条变化。关键日期从有值变为空属于事实字段更新，并计入变化条数。

### 6.4 最小同步状态

数据库只持久化一个订单融资自动同步状态，不建立导入报告或历史明细表：

- 最近成功时间。
- 最近成功时的实际变化条数。
- 最近成功源文件版本或哈希，用于跳过完全相同的文件。
- 最近尝试的计划时点，用于防止同一时点重复执行。

失败原因只进入服务端结构化日志，不通过订单融资页面展示。

## 7. 安全与权限

- 自动同步仅申请和调用读取文件所需能力，不需要删除权限。
- WPS凭据不得写入Git、数据库业务字段、前端或日志。
- 手动上传继续受订单融资敏感导入权限控制。
- 自动同步以服务身份运行，不扩大普通用户页面权限。
- 本次先在 Staging Render和Staging Supabase验证；Production环境变量、数据库变更和正式部署必须单独确认。

## 8. 验收标准

### 8.1 WPS联通

- 使用真实共享文件成功读取元数据并下载有效 Excel。
- 能识别`订单`、`额度`、`预警`，其他页签不影响导入。
- 同一文件重复同步时变化条数为0。

### 8.2 阶段和风险

- 无人工装船、无交单日时按最迟装船日产生高/中风险。
- 人工确认装船后进入`已装船待交单`并清除装船风险。
- WPS出现交单日后自动认定装船、进入`已交单待回款`并清除装船和待交单风险。
- 未交单且到达最早交单截止日时产生中风险并进入本周重点。
- 未回款且回款到期不足等于7天或逾期时为高风险，8至30天为中风险。
- 多笔融资部分回款仍为`已交单待回款`并显示部分回款；全部回款后为`已回款待结案`。
- WPS删除交单日或回款日后状态和风险正确回退，人工装船确认保留。
- WPS明确结案后为`已完成`，不参与存续风险和金额统计。

### 8.3 页面

- 页面加载后可见最近成功同步时间和实际变化条数。
- 卡片、筛选和指标使用新的阶段名称及“回款”术语。
- 主卡片不再显示独立确认状态和展期状态。
- 单笔融资展开不重复主卡片字段；多笔融资逐笔信息完整。
- 管理员可以确认和撤销装船，普通只读用户看不到操作按钮。

### 8.4 调度和失败边界

- 测试时钟下09:00和17:00各触发一次，同一计划时点不重复。
- WPS下载失败、授权失效、Excel解析失败或数据库写入失败时，现有订单数据和上次成功同步状态保持不变。
- 服务端日志包含失败阶段和可定位错误，但不包含凭据、token或下载地址。

## 9. 不在本次范围

- 不建设第三个订单融资页面。
- 不建设导入报告、同步历史、异常明细或报告导出。
- 不增加人工交单或人工回款按钮。
- 不记录或判断集港状态，不区分FCR。
- 不订阅WPS实时事件。
- 不修改融资资金监控的额度业务口径，只保证其继续读取同步后的有效订单数据。
- 不操作Production配置、数据库或部署。
